1. CORE BACKEND
===========================================

Framework:
-----------

Why:
----
- Extremely fast development speed
- Automatic Swagger/OpenAPI documentation
- Built-in async support
- Easy validation with Pydantic
- Excellent for REST APIs
- Cleaner than Flask for enterprise APIs
- Judges instantly see professional API docs

Used For:
---------
- REST API layer
- Middleware orchestration
- Adapter execution
- Authentication
- Health checks
- Metrics endpoints



Language:
---------
Python 3.11+

Why:
----
- Fastest language for hackathon productivity
- Huge ecosystem for parsing XML/CSV/SOAP
- Easy AI integrations
- Easy async programming
- Strong data transformation libraries



===========================================
2. LEGACY SYSTEM SIMULATOR
===========================================

Purpose:
--------
Simulate old enterprise systems with ugly data formats.

Components:
-----------
- CSV dump reader
- XML/SOAP service
- Fixed-width text parser
- Random failure generator

Why:
----
Judges need to SEE the pain/problem.

We intentionally create:
- inconsistent schemas
- malformed responses
- random downtime
- strange field names
- weird date formats

This creates dramatic before/after transformation.



Tech Used:
----------
- Python scripts
- Flask or FastAPI mini services
- Static files
- Randomized response corruption
